12.1

Separating Hyperplanes

In binary classification, we have data points $\mathbf{x}_n \in \mathbb{R}^D$ with labels $y_n \in \{+1, -1\}$. Our goal: find a function that correctly predicts labels for new data.

Definition: Linear Classifier

A linear classifier is defined by a hyperplane:

$$f(\mathbf{x}) = \langle \mathbf{w}, \mathbf{x} \rangle + b = \mathbf{w}^\top \mathbf{x} + b$$

where $\mathbf{w} \in \mathbb{R}^D$ is the normal vector (perpendicular to the hyperplane) and $b \in \mathbb{R}$ is the bias (intercept).

Classification rule: predict $+1$ if $f(\mathbf{x}) \geq 0$, else predict $-1$.

The hyperplane $\{\mathbf{x} : f(\mathbf{x}) = 0\}$ separates the feature space into two half-spaces ‚Äî one for each class.

Correct Classification Condition

For a training example $(\mathbf{x}_n, y_n)$ to be correctly classified:

$$y_n \cdot f(\mathbf{x}_n) = y_n(\mathbf{w}^\top \mathbf{x}_n + b) > 0$$

This elegant form works because:

  • If $y_n = +1$ and $f(\mathbf{x}_n) > 0$ ‚Üí product is positive √¢≈ì‚Äú
  • If $y_n = -1$ and $f(\mathbf{x}_n) < 0$ ‚Üí product is positive √¢≈ì‚Äú
  • Misclassification ‚Üí product is negative √¢≈ì‚Äî
12.2

The Margin

For linearly separable data, infinitely many hyperplanes can separate the classes. The SVM chooses the one that maximizes the margin ‚Äî the distance to the closest training points.

Definition: Margin

The margin is the perpendicular distance from the hyperplane to the nearest training point(s):

$$\text{margin} = \frac{1}{\|\mathbf{w}\|}$$

(when normalized so the closest points satisfy $|f(\mathbf{x})| = 1$)

Maximum Margin Optimization

The SVM finds the hyperplane by solving:

$$\min_{\mathbf{w}, b} \frac{1}{2}\|\mathbf{w}\|^2 \quad \text{subject to} \quad y_n(\mathbf{w}^\top \mathbf{x}_n + b) \geq 1, \; \forall n$$
Why Maximize Margin?

A larger margin provides better generalization ‚Äî the classifier is more robust to noise and new data. Intuitively, it leaves more "breathing room" around the decision boundary.

Support Vectors

The support vectors are the training points that lie exactly on the margin boundaries ($y_n f(\mathbf{x}_n) = 1$). They "support" the hyperplane:

  • Only support vectors matter ‚Äî other points could be removed without changing the solution
  • The optimal $\mathbf{w}$ is a linear combination of support vectors
  • Typically only a small fraction of training data are support vectors

12.3

Soft Margin SVM

Real data is rarely perfectly linearly separable. The soft margin SVM allows some points to violate the margin constraints, using slack variables.

Soft Margin Formulation
$$\min_{\mathbf{w}, b, \boldsymbol{\xi}} \frac{1}{2}\|\mathbf{w}\|^2 + C \sum_{n=1}^{N} \xi_n$$

subject to:

$$y_n(\mathbf{w}^\top \mathbf{x}_n + b) \geq 1 - \xi_n, \quad \xi_n \geq 0$$

where $\xi_n$ are slack variables and $C > 0$ is the regularization parameter.

Interpreting the Slack Variables

  • $\xi_n = 0$: Point is correctly classified and outside margin
  • $0 < \xi_n < 1$: Point is correctly classified but inside margin
  • $\xi_n \geq 1$: Point is misclassified

The C Parameter

C Value Effect Trade-off
Large C Penalize violations heavily Smaller margin, fewer errors on training data
Small C Allow more violations Larger margin, more tolerant of outliers
Why This Matters in ML

The C parameter controls the bias-variance trade-off:

  • Large C: Low bias, high variance (risk of overfitting)
  • Small C: High bias, low variance (risk of underfitting)

Use cross-validation to find the optimal C!

12.4

The Kernel Trick

What if data isn't linearly separable in the original space? Map it to a higher-dimensional space where it becomes separable!

Feature Map

A feature map $\phi: \mathbb{R}^D \to \mathbb{R}^M$ transforms data into a higher-dimensional space:

$$\mathbf{x} \mapsto \phi(\mathbf{x})$$

Then we find a linear separator in the $\phi$-space:

$$f(\mathbf{x}) = \mathbf{w}^\top \phi(\mathbf{x}) + b$$

The Kernel Trick

Computing $\phi(\mathbf{x})$ explicitly can be expensive (or impossible if $M = \infty$). The kernel trick avoids this by computing inner products directly:

$$k(\mathbf{x}_i, \mathbf{x}_j) = \langle \phi(\mathbf{x}_i), \phi(\mathbf{x}_j) \rangle$$

Popular Kernels

Kernel Formula Use Case
Linear $k(\mathbf{x}, \mathbf{x}') = \mathbf{x}^\top \mathbf{x}'$ Linearly separable data
Polynomial $k(\mathbf{x}, \mathbf{x}') = (\mathbf{x}^\top \mathbf{x}' + c)^d$ Polynomial decision boundaries
RBF (Gaussian) $k(\mathbf{x}, \mathbf{x}') = \exp(-\gamma\|\mathbf{x} - \mathbf{x}'\|^2)$ Complex, nonlinear boundaries
Example: XOR Problem

The XOR pattern (points at (0,0), (1,1) are class +1; points at (0,1), (1,0) are class -1) is not linearly separable in 2D.

But with the polynomial kernel $k(\mathbf{x}, \mathbf{x}') = (\mathbf{x}^\top \mathbf{x}')^2$, the implicit feature map includes the term $x_1 x_2$, which makes the problem separable!

Real-World Applications
  • Image classification: Histogram intersection kernels
  • Text classification: String kernels for spam detection
  • Bioinformatics: Sequence kernels for protein classification
  • Handwriting recognition: RBF kernels for digit classification
